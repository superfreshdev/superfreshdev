Core Feature's (07)
Login & Registration
Login goes over Keycloak Authentificaiton ( Security Hashing Concpet, better than classic AES 256 ). Registration goes with E-Mail Verifaction and after it will create an new user in the postgreSQL - Database. The Layout was created by Angular, CSS, SASS with the Keycloak Theming Feature. After Login it goes to the Angular Frontend which runs in an Spring Boot Backend Application.
4 Microservices:
Every Microservcies run in an Docker Container in an Kubernetes-Cluster. There are 2 identic Pods ( Kubernetes ) to handle fallback if one Pod is crashing down.
To let 100.000 User connecting in the same time to this 4 Microserivces there exist an own Load Balancer Pod. Bettwen there we haven API Gateway to handle more security user communication in the system.
User Dashboard:
After Login the user see his User Web Dashboard where he can see his auctally insurances, new activities from the insurance company what the user had to do, for example the company need data for an exident to help customer on the financial way.
The User can see more attractive and extend insurance offer he or she can add to his actually Ã­nsurance contracts. It comes with an prototyp AI Chatbot where the user can write or talk with the AI to help and give all answers about insurance and finance only in the insurance company context.
Its like chatgbt but only smaller.
